# CGA-v2 新仓库 AAAI 投稿执行计划：先生成 NUDT 证据，再写论文

## 0. 结论

你这次判断是对的：**如果当前代码是全新的 overlay/release 仓库，而且没有任何实验产物，那么不能把旧主仓库 frozen artifacts 当成当前代码的论文证据。**

因此，上一版“双轨计划”需要改成：

```text
AAAI paper evidence must be generated by the current new codebase.
旧主仓库结果只能作为历史参照 / target trend，不能作为当前新代码的 paper evidence。
```

正确路线不是：

```text
旧主仓库 frozen evidence = 论文主证据
新 overlay = 仅发布验证
```

而是：

```text
新代码仓库 = 当前论文证据源
必须先在新仓库生成 NUDT-SIRST seed42/43/44 artifacts
再写 ablation / failure case / manuscript tables
```

这与你上传的 narrow-claim rescue plan 一致：该计划明确说当前不能按“新 SOTA 模型 + 多 benchmark validation”直接投，路线应是 `revise / narrow claim / evidence-first`，并且由于新文件夹没有实验产物，第一优先级是重新生成 NUDT-SIRST 主数据集证据。

## 4. 代码结构要求

当前新仓库应具备完整 repo-grade 结构，而不是单文件模型。

必须覆盖：

```text
model/
  CGA_MSHNet.py
  cga_aux.py

utils/
  cga_targets.py

loss.py
net.py
train.py
test.py
evaluate.py
metrics.py

tools/official/
scripts/official/
tests/
docs/internal/
docs/paper/
configs/
```

### 4.1 model contract

`model/CGA_MSHNet.py`：

```text
train mode:
  returns final_logit + aux_outputs

eval/test mode:
  returns final_logit only
```

`model/cga_aux.py`：

```text
center head
boundary head
scale head
peak head
```

`utils/cga_targets.py`：

```text
GT mask -> center / boundary / scale / peak targets
```

### 4.2 loss contract

`loss.py`：

```text
MSHNetCGALoss = base segmentation/OHEM loss + CGA auxiliary loss
```

要求：

```text
auxiliary target generation 不写在 train loop 里
loss finite
loss keys stable
aux flags explicit
lambda values written into summary/config
```

### 4.3 eval contract

`test.py / evaluate.py / inference.py`：

```text
only final_logit used
probability = sigmoid(final_logit)
threshold = 0.5
no auxiliary head inference
no post-processing
no TTA
no checkpoint averaging
```

### 4.4 summary identity contract

每个 train/eval summary 必须写：

```json
{
  "model": "MSHNetCGA or MSHNetOHEM",
  "dataset": "NUDT-SIRST",
  "seed": 42,
  "checkpoint_epoch": 400,
  "threshold": 0.5,
  "dataset_root": "...",
  "train_list_sha256": "...",
  "test_list_sha256": "...",
  "config_sha256": "...",
  "code_commit_or_overlay_hash": "...",
  "summary_created_by": "..."
}
```

否则 checker 不能只读 metric，必须先校验身份。
