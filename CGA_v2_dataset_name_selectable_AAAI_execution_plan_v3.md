# CGA-v2 Dataset-Name-Selectable AAAI Execution Plan v3

## 0. Decision

This document revises the previous dataset-name-selectable plan. The correct engineering direction is **not** to create separate hard-coded versions for NUDT-SIRST, NUAA-SIRST, and IRSTD-1K. The repository should follow a BasicIRSTD-style dataset-selection interface:

```bash
DATASET_DIR=/path/to/datasets \
DATASET_NAME=NUDT-SIRST \
SEED=42 \
bash scripts/official/run_cga_v2_dataset_one_seed_train_eval.sh
```

Only `DATASET_NAME` changes across datasets:

```bash
DATASET_NAME=NUDT-SIRST
DATASET_NAME=NUAA-SIRST
DATASET_NAME=IRSTD-1K
```

The current AAAI route remains:

```text
revise / narrow claim / evidence-first
```

CGA-v2 should be framed as:

```text
Target-preserving component-geometry regularization for MSHNet-style IRSTD.
```

It should **not** be framed as a new SOTA model, complete hard-clutter closure, transfer learning, or multi-benchmark validation before the corresponding evidence exists.

## 1. Core Rules

### 1.1 New repository does not borrow old results

If this is a new overlay / repo-grade implementation, then historical OHCM-MSHNet-main results are **sanity references only**. They are not current-repo paper evidence.

Required statement:

```text
This repository is a new repo-grade implementation.
Historical OHCM-MSHNet-main results are internal sanity references only.
All paper evidence used by this repository must be regenerated by this repository.
```

Output:

```text
docs/internal/cga_v2/repo_identity/code_delta_statement.md
```

### 1.2 NUDT-SIRST first

The execution order is:

```text
P0: repo contract + code identity statement
P1: dataset registry + NUDT-SIRST preflight
P1.5: closest prior-art threat table
P2: NUDT-SIRST seed42 reproduction decision rule
P3: NUDT-SIRST seed43/44 multiseed reproduction
P4a: seed42 ablation required for submission
P4b: multiseed ablation not required
P5: failure / diagnostic pack
P6: closest baseline / prior-art pack
P7: claim linter + manuscript pack
```

Do **not** start NUAA-SIRST or IRSTD-1K before the NUDT-SIRST main evidence is regenerated.

### 1.3 No model tuning during evidence regeneration

Forbidden during this phase:

```text
architecture tuning
loss tuning
threshold search
checkpoint search
seed search
ablation winner promotion
DPS rescue
post-processing
verifier head
new hard-clutter suppression branch
```

Allowed:

```text
repo contract checks
dataset registry and preflight
fixed OHEM vs CGA paired training/evaluation
predeclared gate checking
implementation audit if seed42 fails
paper evidence packaging
```

### 2.3 Why `expected_counts` should be null initially

Do not hard-code counts such as:

```yaml
expected_counts:
  train: 697
```

unless the current split file has been verified in this repository. The correct initial schema is:

```yaml
expected_counts:
  train: null
  test: null
  hcval: null
```

The preflight summary will record the observed counts and the split hashes. If a later paper package wants to freeze expected counts, it should do so from the generated preflight artifact, not from memory.

## 3. Dataset Registry and Preflight Code

### 3.1 Add registry utility

Add:

```text
tools/official/dataset_registry.py
```

Responsibilities:

```text
load configs/datasets.yaml
resolve DATASET_DIR / DATASET_NAME into dataset root
resolve split files
resolve image/mask paths from list items
apply item_format rules
read mask_policy
return a DatasetSpec object
compute dataset_spec_sha256
compute train/test/hcval list hashes
```

Do not place this lightweight registry inside `utils/`, because importing `utils` can trigger training dependencies. Keep it under `tools/official/` or `tools/common/`.

### 3.2 Add generic preflight

Add:

```text
tools/official/check_cga_v2_dataset_preflight.py
scripts/official/run_cga_v2_dataset_preflight.sh
tests/test_cga_v2_dataset_registry.py
tests/test_cga_v2_dataset_preflight.py
```

The preflight must check:

```text
split file exists
image exists
mask exists
image readable
mask readable
image/mask size match
mask values satisfy mask_policy
empty-mask count
malformed list items
path traversal
train/test overlap
duplicate items
```

### 3.3 Required preflight summary fields

The summary must include identity fields:

```json
{
  "gate": "Gate-CGA-v2-dataset-preflight",
  "dataset": "NUDT-SIRST",
  "dataset_dir": "/path/to/datasets",
  "dataset_root": "/path/to/datasets/NUDT-SIRST",
  "dataset_spec_sha256": "...",
  "train_list_sha256": "...",
  "test_list_sha256": "...",
  "hcval_list_sha256": "...",
  "train_count": 0,
  "test_count": 0,
  "hcval_count": 0,
  "train_test_overlap_count": 0,
  "duplicate_item_count": 0,
  "mask_policy": {
    "type": "binary",
    "allowed_values": [0, 255],
    "canonical_required": false
  },
  "gate_pass": true,
  "decision": "DATASET_PREFLIGHT_PASS"
}
```

Split hashes are required because a registry file can remain unchanged while a split file changes.

## 9. P2: NUDT-SIRST Seed42 Reproduction Decision Rule

### 9.1 Naming

Do not call this gate:

```text

```

Use:

```text
seed42 reproduction decision rule
```

Required statement:

```text
The thresholds are pre-declared before seeing new-repo seed42 results.
```

### 9.2 Summary must include thresholds

The seed42 summary must include:

```json
{
  "decision_rule_predeclared": true,
  "decision_rule_name": "seed42 reproduction decision rule",
  "thresholds": {
    "full_delta_mIoU_min": 0.02,
    "full_delta_precision_min": 0.01,
    "full_delta_pd_min": -0.001,
    "full_delta_fa_ppm_max": 0.0,
    "hcval_delta_miou_min": 0.0,
    "hcval_delta_pd_min": -0.001
  }
}
```

### 9.3 Fixed protocol

```text
DATASET_NAME = NUDT-SIRST
SEED = 42
checkpoint = epoch400
threshold = 0.5
baseline = MSHNetOHEM
candidate = MSHNetCGA
```

### 9.4 Command

```bash
DATASET_DIR=/home/ly/AAAI/OHCM-MSHNet-main/datasets \
DATASET_NAME=NUDT-SIRST \
SEED=42 \
bash scripts/official/run_cga_v2_dataset_one_seed_paired_train_eval.sh
```

### 9.5 If seed42 fails

Do not immediately stop the method. Allow one implementation audit:

```text
P2_FAIL_IMPL_AUDIT_ALLOWED:
  dataset split
  seed determinism
  model registry
  OHEM/CGA loss config
  checkpoint epoch
  eval threshold
  train/eval mode
  summary identity
  final-logit-only evaluation
```

If the audit finds an implementation error, fix only that error and rerun seed42 once.

If no implementation error is found:

```text
STOP_NEW_REPO_CGA_V2_AT_NUDT_SEED42
```

Do not tune the model/loss/threshold/checkpoint.
